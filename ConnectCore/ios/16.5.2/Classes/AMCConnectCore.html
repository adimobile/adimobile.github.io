  • login(_:username:password:scope:successClosure:errorClosure:)

    Login the user with email address and password.

  • Parameters:

    • validatorId: Authentication validator to be used.
    • username: Username.
    • password: Password of user account
    • scope: Required scope
    • successClosure: Closure that will be called if the response from backend is successful. It will pass:

      • requestIdentifier: Identifier for your request. You’ll receive the same identifier when login is executed.
    • errorClosure: Closure that will be called if the response from backend is not successful. It will pass:

      • requestIdentifier: Identifier for your request. You’ll receive the same identifier when login is executed.
      • error: Instance with information about the error. You can receive system errors like No host is available or errors with domain like MOBILE_ERR_0001 when an unexpected response from backend is received or error like invalid_grant when credentials are wrong, or the age of the user does not comply with the legal age of the country

    • Returns

    Identifier for your request. You’ll receive the same identifier when operation is completed.

  • handleResetPassword(_:communicationLanguage:expiryTime:baseUrl:source:successClosure:errorClosure:)

    Triggers a new request to SCV for reset password. User will get an email on the specified email address with instructuctions for password reset.

  • Parameters:

    • email: email where reset password instructions will be delivered.
    • communicationLanguage: Preferred language of communication. Optional.
    • expiryTime: The duration(in minutes) till which reset password token remains valid E.g. 120 = 2 hours.
    • baseUrl: The url where user will set a new password.
    • source: The exact source of touch point of user to SCV. Optional.
    • successClosure: Closure that will be called if the response from backend is not successful.
    • errorClosure: Closure that will be called if the response from backend is not successful.

    • Returns

    Request identifier for tracking purposes.
